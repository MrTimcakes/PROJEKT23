During the development of the “PROJEKT23” website, specifics of the HTML5 and CSS3 standards was analysed to gain an understanding for the best practices for these new standards. HTML5 brings some new tags to better structured pages. For example, instead of pages being never-ending nests of “<div>” elements, there are now “<section>” and “<article>” tags which make it every easy to separate the different portions of the page up into distinct section tags. This is in-action on almost every page of the PROJEKT23 website. HTML5 also bring “<nav>” and “<footer>” tags which helps differentiate a regular div from the pages navigation tools.

CSS3 brings one of my personal favourite features to the table. Flexbox. If you’ve got a problem styling the web, a flexbox can probably fix it. It’s such a versatile new syntax, simply display a parent object as a flexbox and you can then both vertically and horizontally centre its children, in CSS2 is this was no easy feat, not impossible, but definitely not 3 lines like CSS3 with the power of flexbox. But flexbox’s capabilities far extend centring things, flexbox lets you positions elements pretty much however you want, want your items to fill a space but have an equal gap between them? Flexbox can do that. Need a list that displays horizontally when there’s enough space? Yep, Flexbox.

Media queries are a huge part of the modern web, media queries are revolutionary feature of CSS3, they allow different styles to be set deepening on a characteristic like browser viewport size. This enables web-developers to enable a single webpage to be tailor designed to fit a multitude of different screen sizes. This is vitally important as the majority of web traffic in todays era is mobile sites. CSS3 really brought web back into the limelight and shows it’s a really serious platform.

It’s not only HTML and CSS that have had some major overhauls. JavaScript has been evolving along with them. The latest standard of JavaScript ES8 or ES2017 has a huge range of functionality over previous “last-gen” standards. The language has had features and browser APIs added, but It’s even had Syntax changes to the actual language. JavaScript can now support classes like a real Object-Oriented Language, its now got promises and even asynchronous functions. The world really is your Oyster with these new standards. For the PROJEKT23 site I took advantage of a few of these new features, notably the Canvas/WebGL and LocalStorage APIs. First LocalStorage, it’s similar, ish, with the good old generic cookie, though it is far more versatile and safe. LocalStorage is an API for developers to use to store, actually a quite large amount of data on the client’s computer, safe from anyone else’s prying eyes. LocalStorage content is locked down to the site that created it, its not accessible any other way. In my project I used LocalStorage to store the Cart information of the users shopping and the latest state of their customisation so when they return the can pick right up where they left off. Now for the Canvas/WebGL, this tool allows JavaScript to do some impressive visual stuff, especially the WebGL, that is really performance efficient. These are two APIs that give your JavaScript direct access to a, well, “canvas”. Your scripts can write on that canvas however they please. In my case I used a library “ThreeJS” to load a real 3D model, the same type that would be used in desktop software like blender or 3DSMax (.obj) and display it to the user. This is all processed at above 60 frames per seconds and so makes a very fluid user experience. In the PROJEKT23 customiser the user can change the decal on the under side of the board and change the colour of the wheels to their personal preference. However, this isn’t all great, because the WebGL API has the potential to be so powerful, it just cannot be user friendly and is a very difficult API to use, that is why I chose to use the ThreeJS library, to make it manageable.

For my Navbar I used a CSS “Hack” which involves using a label to toggle a hidden checkbox and then using styles with CSS’s “~” sector which matches item after it in the DOM. This allowed me to make a super-efficient navbar which is pure CSS, no JavaScript required. I also used a Pre-Processor, SCSS which compiles into regular CSS, this was just to make my life easier with nesting in CSS.

W3C Validation

The W3C (World Wide Web Consortium) validator is a HTML mark-up checker, it goes through and makes sure tags are closed, images have alternate text and also checks for some best-practise semantics. I tested the PROJEKT23 website against the validator at https://validator.w3.org/nu/. Every page passed, however there was one or two warnings that sections weren’t followed by a header, this is because I used the section tag to enforce structure in the mark-up and not-necessarily for what a user would see.
